OpenRelix™

OpenRelix 意为开源的个人记忆珍藏：把每天和 AI CLI 一起做过的工作，沉淀成可复用的技能、方法手册、模板、自动化、记忆和可视化复盘，而不是散落在聊天记录里。

查看功能导览 安装方式

当前预览版：产品定位不绑定单一 AI host；当前首个公开适配器是 Codex CLI 和 Codex app-server，Claude Code、Gemini CLI 等其他 AI CLI / coding-agent host 在后续适配计划中。本页预览使用脱敏示例面板，不包含真实项目、账号、路径、日志或工作内容。

痛点

AI 编程越来越强，但个人经验和记忆仍然很容易流失。

一次很有价值的排障、一次高质量代码审查、一次顺手写出的脚本，如果没有结构化沉淀，过几天就只剩“我好像做过”。

聊天碎片

工作成果留在会话里

复盘、规则、排障路径和模板散落在不同窗口，想复用时需要重新翻上下文。

记忆失控

AI host 记忆不可视

AI host 原生记忆很有用，但很难直接比较、盘点和判断哪些记忆真正带来了复用价值。

隐私边界

个人数据不该跟源码混在一起

可复用能力适合开源，但真实日志、复盘、登记册和运行缓存必须留在本地。

复用困难

OpenRelix 个人记忆需要账本

你需要看见哪些方法最常复用、节省了多少时间、哪些经验值得升级成技能，同时不挤占过多上下文。

它是什么

它是 AI CLI 的本地个人资产层，不是另一个云端知识库。

仓库只放可复用能力；用户自己的运行状态写到独立状态目录。默认开启本地个人记忆，当前 Codex 适配器只把压缩后的有界摘要同步进 host 原生上下文。

源码仓库

可开源能力层

安装器、技能、模板、LaunchAgent 模板、自动化脚本和产品文档都能公开审阅。

状态目录

本地个人数据层

复盘、登记册、原始窗口、汇总摘要、报表和运行缓存都保存在用户机器。

上下文策略

上下文轻量可控

默认 integrated：本地记忆入库，轻量摘要进入当前 host 上下文；需要隔离时再显式使用 local-only 或 off。

个人归属

资产永远归个人

方法手册、技能、模板、自动化和记忆账本不绑定单一平台，未来可迁移到 Claude Code、其他 CLI 或团队工作流。

记忆系统

先沉淀成本地账本，再把必要摘要交给 AI host。

OpenRelix 不会把所有聊天都塞回上下文。它先把工作记录留在本机，再按固定节奏整理：平时每 30 分钟做小结，深夜做当天总账；需要时也能手动立刻生成。

先把这段工作记下来从 AI CLI 已保存的会话记录里读取刚做过的窗口，标清日期、项目和主题；Codex 应用里的对话只有你开启后才会读。
每 30 分钟做一次小结开启后台学习刷新后，它会每 30 分钟看一眼最近窗口，把新结论、待跟进事项和可复用做法整理进本地记忆和面板。
深夜把一天收成总账晚上 23:00 先整理当天窗口，00:10 再把前一天结果固定下来，生成长期记忆、短期跟进、报告和面板。
着急时手动跑一次不想等后台任务时，直接运行 review 或 refresh，就能马上从当前窗口生成记忆；默认只同步压缩摘要，local-only 时只留本地。
默认模式

本地记忆 + 轻量上下文

完整记忆账本留在本机；进入 host 的只是压缩摘要，目标 6.7K、上限 8K token，避免长期上下文膨胀。

可追溯

每条记忆都有来源

记忆会保留来源窗口、日期、项目上下文和隐私边界，方便之后回看为什么沉淀、是否还值得保留。

可升级

重复经验会变成资产

多次复用的排障路径、检查清单或提示词会进入资产候选，后续可以升级为 skill、playbook、template 或 automation。

功能导览

从一天的 AI agent 工作，到可复用资产面板。

下面的预览是脱敏示例面板：保留真实布局、交互密度和模块结构，内容替换为示例工作区和示例数值。

面板

每天自动把工作变成资产

AI agent 窗口结束后，OpenRelix 把当天的结论、下一步、可复用做法和隐私状态整理进一个本地面板。

  • 今天做了什么、沉淀了什么、还缺什么，一屏能扫完。
  • 重复出现的流程会标成技能、模板或自动化候选。
  • 只存脱敏摘要和来源索引，个人状态默认留在本机。
  • 输出 HTML、Markdown、JSON、CSV，方便本地查看或二次加工。
复盘

立刻分清模型记忆和个人资产账本

左边是会影响下次回答的 host 原生记忆；右边是 OpenRelix 本地账本，记录来源、状态和可升级动作。

  • 稳定偏好和通用方法可以进上下文，临时任务默认留本地。
  • 每条 OpenRelix 记忆带来源、分桶和更新时间，方便回溯核对。
  • 重复工作可以从候选升级为技能、模板或自动化。
记忆模式

上下文只放该放的内容

OpenRelix 会把窗口先按项目和主题收拢，再决定哪些进模型上下文、哪些只留在本地资产账本。

  • 默认 integrated：完整登记册留在状态目录，压缩摘要进入当前 host 原生上下文。
  • `--use-integrated`：显式选择 host 上下文注入模式。
  • `--record-memory-only`：只做本地记录和可视化，不注入模型上下文。
  • 压缩策略：同签名记忆跨天归并，优先长期和任务记忆，低优先级只留本地；摘要目标 6.7K、上限 8K token。
流程

流程很短，自动化可控。

首次使用按快速上手来：先整理最近 7 天，再打开本地面板看结果。它不靠实时 hook 打断工作，而是本地离线采集、按需刷新、定时整理、可视化查看。

采集后台刷新或手动脚本读取当前 host 历史和会话元数据，按日期和窗口归档。
打开补齐最近 7 天后运行 `openrelix open panel`，查看本地账本和同步状态。
分类手动复盘和夜间整理都会区分长期、任务、低优先级记忆，减少长期记忆噪声。
登记把可复用内容记录到 JSONL 登记册，保留类型、范围、复用价值和来源。
可视化后台刷新、夜间任务或手动脚本生成概览、面板和 CSV，让资产是否有用变得可见。

安装方式。

推荐安装

一行命令安装。

已经有 Node.js 18+ 和 Python 3.10+ 的 macOS 用户，直接跑这一行。它会安装 `openrelix` 命令、创建本地资产账本、开启每 30 分钟的增量学习，并读取 Codex CLI 和 Codex 应用的本地记录。

第三步（可选）

查看或切换 OpenRelix 记忆模式

`integrated` 是默认安装档位和默认记忆模式；后续只在需要严格隔离或只做可视化时才用 `openrelix mode` 切换。

OpenRelix 记忆全开模式（默认） 生成 OpenRelix 本地记忆文件和资产面板；完整 registry 留在 state root，压缩摘要注入当前 host 上下文。
openrelix mode integrated
OpenRelix 记忆本地存储模式 仅生成和存储 OpenRelix 本地记忆与面板，不做上下文注入。
openrelix mode local-only
禁用 OpenRelix 记忆模式 不生成 OpenRelix 记忆文件，不写 OpenRelix 本地 memory registry；本项目仅做当前 host 的资产可视化。
openrelix mode off
全开本地记忆 + 上下文注入
本地存储只写本地，不注入
禁用只做资产可视化

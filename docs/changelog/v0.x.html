v0.2.0 2026-04-29 已发布

桌面端来了，面板会自己提醒升级

v0.1 系列之后的第一次 minor 升级。OpenRelix 不再只是命令行——它现在是个能在 Launchpad 里搜到的 macOS app，并且会在右下角小声提醒你「有新版了」。

新功能
  • macOS 桌面端：真实 app bundle 自动同步到 ~/Applications/OpenRelix.app，Launchpad / Spotlight / Finder「应用程序」更稳定可发现，不用再翻终端
  • 面板内自动检查新版本：右下角飘个小药丸告诉你有新版可升，客户端面板可以一键执行 openrelix update --yes（走本地 Token 校验 + file:// 来源限制，杜绝公网页面摸进来）；浏览器端没有本地权限，降级为帮你把 npm update -g openrelix 复制到剪贴板
  • 安装 / 更新结束后会主动问你两句：要不要现在就学一下最近 7 天的记忆（5–15 分钟）？要不要顺手打开客户端？不想被问就加 --no-learn / --no-launch，CI 和非交互终端自动跳过
优化
  • codex-native 中文渲染规则全部搬到 <state_root>/personal_codex_rules.py，仓库引擎不再带任何示例数据；升级后少数原生记忆话题会回退到英文原文，需要的话自己往个人规则文件里加就回来了
  • 加了发布前 personal-info 自动检查：CI 卡口 + 可选本地 pre-commit hook，扫描跟踪文件里的用户路径 / 个人邮箱 / 硬编码凭据，以及你自己写的 <state_root>/personal_denylist.txt 模式
  • 安装时语言选择的中英文提示都强调了一遍：选完之后 UI、本地记忆、模型生成内容会一起锁定语言，事后想切得重装并清空记忆
  • 面板上 Token 账单做了四舍五入，不再拖一长串小数
  • 发布流程切到 npm trusted publishing GitHub Actions，少踩一步手动 token 的雷
修复
  • 修了一个隐蔽但影响很大的坑：后台 LaunchAgent 跑 nightly 时找不到 codex 二进制，被迫回退到 history 模式，结果 VS Code 扩展和那些只活在 app-server 里的对话整窗口丢失。现在 plist 会注入 CODEX_BIN 和安全 PATH，老用户也兜底，fallback 路径扩展到 volta / bun / nvm
  • review 刷新完面板会跟着更新输出了，不再卡着旧数据
  • Token 实时刷新后账单不再错位

v0.1.0 2026-04-28 首发

OpenRelix 公开预览版首发

跑通本地优先个人 AI 资产系统的最小完整闭环：把每天和 AI CLI 一起做过的工作沉淀成可复用的技能、模板、自动化、记忆和可视化复盘，而不是散落在聊天记录里。

核心能力
  • 一键 npm 安装与 install 脚本，覆盖 macOS 默认目录布局
  • Codex CLI 与 Codex app-server 工作窗口采集，本地优先
  • 任务复盘、资产登记、长期/短期记忆分层
  • 面板可视化：日摘要、关键指标、Token 信号
  • macOS LaunchAgent 后台自动化（每 30 分钟刷新、深夜固化）
边界与隐私
  • 仓库只放可复用能力（安装器、技能、模板、自动化脚本、文档）
  • 个人状态默认写到独立状态目录，不进入 npm 包和源码仓库
  • 密钥、Token、Cookie、原始内部日志不进入资产库与公开预览

# Code of Conduct

## Expected Behavior

Participants should be respectful, practical, and focused on improving the project.

Examples of expected behavior:

- use clear, constructive technical feedback
- respect privacy and local data boundaries
- avoid sharing another person's private context or credentials
- keep discussion relevant to the issue or pull request

## Unacceptable Behavior

Unacceptable behavior includes harassment, threats, discriminatory language, deliberate publication of secrets or private data, and repeated disruption of project discussion.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; block participants; or take other reasonable action to protect the project and its contributors.

Reports should include a concise description, relevant links, and any needed context. Do not include private data unless a maintainer explicitly asks for a sanitized example.
